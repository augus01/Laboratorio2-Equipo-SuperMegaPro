Comandos Docker

Guía de los comandos esenciales de Docker organizados por categoría. Úsala como referencia rápida durante el desarrollo.

01

Gestión de Imágenes

📥

Descargar una imagen

Descarga una imagen desde Docker Hub sin ejecutar ningún contenedor.

Sintaxis docker pull <imagen>:<tag>
Ejemplo docker pull nginx:alpine
📋

Listar imágenes

Muestra todas las imágenes descargadas en tu máquina local.

Comando docker images
💡 También puedes usar docker image ls como alternativa.
🔨

Construir una imagen

Crea una imagen propia a partir de un Dockerfile en la carpeta actual.

Sintaxis docker build -t <nombre>:<tag> .
Ejemplo docker build -t mi-web-html .
🗑️

Eliminar una imagen

Borra una imagen de tu máquina local. La imagen no puede estar en uso por ningún contenedor.

Sintaxis docker rmi <imagen>
⚠️ Detén y elimina primero los contenedores que usen la imagen.
02

Gestión de Contenedores

▶️

Ejecutar un contenedor

Crea y arranca un contenedor a partir de una imagen. Es el comando más usado en Docker.

Sintaxis docker run -d -p <host>:<contenedor> --name <nombre> <imagen>
Ejemplo docker run -d -p 8080:80 --name mi-web mi-web-html
📊

Listar contenedores

Muestra los contenedores activos. Con el flag -a muestra también los detenidos.

Solo activos docker ps
Todos docker ps -a
⏹️

Detener un contenedor

Envía una señal de parada al contenedor para que se detenga de forma ordenada.

Sintaxis docker stop <nombre o ID>
Ejemplo docker stop mi-web-container
🔁

Iniciar un contenedor detenido

Vuelve a arrancar un contenedor que fue detenido previamente sin recrearlo.

Sintaxis docker start <nombre o ID>
Ejemplo docker start mi-web-container
🗑️

Eliminar un contenedor

Borra un contenedor detenido de forma permanente. No elimina la imagen asociada.

Sintaxis docker rm <nombre o ID>
⚠️ El contenedor debe estar detenido. Usa docker stop primero.
💻

Acceder al interior del contenedor

Abre una terminal interactiva dentro del contenedor para inspeccionar o ejecutar comandos.

Sintaxis docker exec -it <nombre> sh
💡 Usa bash en vez de sh si la imagen lo soporta.
03

Logs e Información

📄

Ver logs del contenedor

Muestra la salida estándar del contenedor. Útil para depurar errores en la aplicación.

Sintaxis docker logs <nombre o ID>
En tiempo real docker logs -f <nombre o ID>
🔍

Inspeccionar un contenedor

Devuelve información detallada en JSON sobre la configuración y estado del contenedor.

Sintaxis docker inspect <nombre o ID>
📈

Ver uso de recursos

Muestra en tiempo real el consumo de CPU, memoria y red de los contenedores activos.

Comando docker stats
04

Limpieza del sistema

🧹

Eliminar contenedores detenidos

Borra todos los contenedores que no están en ejecución de una sola vez.

Comando docker container prune
🧹

Eliminar imágenes sin uso

Borra todas las imágenes que no están siendo usadas por ningún contenedor.

Comando docker image prune -a
⚡

Limpiar todo el sistema

Elimina contenedores detenidos, imágenes sin uso, redes y caché de construcción en un solo comando.

Comando docker system prune -a
⚠️ Este comando es irreversible. Revisa bien qué contenedores e imágenes tienes antes de ejecutarlo.
